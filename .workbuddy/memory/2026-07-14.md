## 修复"下载小说章节内容不全和获取失败"

**任务**: 用户反馈下载小说时章节内容不全、章节获取失败

**根因分析**:
1. scraper `fetchFanqieChapterContent` 无重试机制，每种策略只尝试一次就静默跳过
2. `contentMap[cc.title]` 用 title 作 key，同名章节会覆盖导致内容丢失
3. `content.length > 50` 阈值太低，截断内容直接接受无检测
4. `RankingController` 章节索引从 0 开始（`item.setIndex(i)`），与前端 1-based 不一致

**修复内容**:

### Task #21: 重构 scraper 下载逻辑增强健壮性
- `scraper/src/download.js` import 增加 `MOBILE_HEADERS`
- 新增 `UA_POOL` 轮换池（PC / 移动 / 微信内嵌浏览器）
- 新增 `decodeBest()` 辅助函数（对比 mode 0/1 中文字符占比选最优）
- 新增 `isTruncated()` 截断检测：极短(<50)、无句末标点、引号未闭合（已去掉200字硬阈值，191字有结尾标点的短序言不再被误杀）
- 重写 `fetchFanqieChapterContent`：retryCount + bestContent 参数、UA轮换、截断检测、最多3次重试（1s/2s指数退避）
- API策略新增 JSON 截断容错：检查 `{`...`}` 首尾后再 JSON.parse，避免 `Unexpected end of JSON input`
- 重试失败后返回 bestContent（已有最佳内容）而非 null，保证"先返回成功部分"
- 修复 `downloadFanqieBook` 的 contentMap：从 `cc.title` 改为 `cc.chapterId` 作 key，避免同名覆盖
- batchRun 回调包裹 try-catch，确保异常时也返回结构化结果而非 Promise.reject

### Task #22: 修复下载章节索引错位
